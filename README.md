# ☕ Coffee Shop Sales Dashboard (Power BI)

## 📌 Project Overview
This project is an interactive **Power BI dashboard** designed to analyze **coffee shop sales performance** using transactional sales data.  
The dashboard focuses on tracking **sales, orders, and quantity sold**, while providing insights across **time, product categories, store locations, and customer purchasing patterns**.

The goal of this project is to demonstrate **business-focused analytics**, clean KPI design, and effective data storytelling using Power BI.

---

## 📊 Key Metrics Displayed
Based on the dashboard visuals, the following KPIs are tracked:

- **Total Sales**: `$13K`
- **Total Orders**: `2,816`
- **Total Quantity Sold**: `4,111`
- **Month-over-Month (MoM) Growth**: ~`18%` increase vs Last Month (LM)

Each KPI includes a **trend indicator** to quickly evaluate performance changes compared to the previous month.

---

## 🎯 Business Questions Answered
- How are **sales and orders trending over time**?
- Which **products and categories** generate the highest revenue?
- How does performance vary by **store location**?
- What is the difference between **weekday and weekend sales**?
- Which **hours and days** generate the most sales?

## 🧩 Data & Modeling Assumptions
- Data represents **coffee shop transactional sales**
- Time-based fields include **date, day, and hour**
- Products are grouped into **categories and product types**
- Measures are created using **DAX** inside the Power BI model
- Data is assumed to be **cleaned and transformed** using Power Query

*(Raw data is embedded within the `.pbix` file)*
